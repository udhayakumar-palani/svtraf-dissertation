# FEW — Framework Execution & Workflows

Centralized repository for all SVTRAF dissertation outputs, artifacts, and generated results.

## Structure

```
FEW/
├── data/           Generated datasets & processed contract data
├── tables/         Statistical results tables & validation outputs
├── figures/        Plots, charts & visualizations
├── models/         Serialized model objects & scoring matrices
└── README.md       This file
```

---

## Data Folder (`data/`)

### `svtraf_scored.csv`
- **Description:** All 150 contracts with computed SVTRAF scores
- **Columns:** contract_id, category, source, ISC, ESC, AMP, INT, SVTRAF_score, financial_loss_usd
- **Generated by:** `Notebook_3_Scoring_Implementation.ipynb`
- **Use case:** Validation against ground truth, correlation analysis

---

## Tables Folder (`tables/`)

### `table_validation_results.csv`
- **Description:** Summary of statistical validation metrics
- **Contents:** NDCG, Spearman ρ, MAE, Wilcoxon test p-value
- **Generated by:** `Notebook_4_Statistical_Validation.ipynb`
- **Use case:** Performance metrics comparison, thesis results section

---

## Figures Folder (`figures/`)

*Reserved for PNG/SVG plots generated from notebooks:*
- Vulnerability category distributions
- SVTRAF vs CVSS scatter plots
- Statistical result visualizations
- Ablation study charts

---

## Models Folder (`models/`)

*Reserved for serialized objects:*
- Fitted scoring models
- Component weight matrices
- Serialized comparison baselines (CVSS, BVSS, Ahmad et al.)

---

## Generation Workflow

To regenerate all artifacts:

```bash
# From repository root:
bash run_notebooks.sh

# Outputs are automatically placed in FEW/ subfolders
```

Expected runtime: ~69 seconds on standard hardware.

## Notes

- All files are **deterministically generated** with seed=42
- Re-running notebooks **overwrites** previous outputs
- Archive important results before re-execution
- Git ignores FEW/ folder (to avoid notebook output bloat in version control)
